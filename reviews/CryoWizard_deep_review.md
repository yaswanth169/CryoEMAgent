## 1. Project Overview

**CryoWizard** is a fully automated single-particle cryo-EM (Electron Microscopy) data processing pipeline that streamlines workflow from raw movies, micrographs, or particles to high-resolution 3D volumes with **zero human intervention**. It targets the cryo-electron microscopy research community for structural biology.

## 2. System Architecture

### Top-Level Design

```
INPUT (Movies/Micrographs/Particles)
  ↓
[IMPORT STAGE] — Load data into CryoSPARC
  ↓
[PREPROCESS STAGE] — Motion correction, CTF estimation, picking/extraction
  ↓
[RANKING STAGE] — CryoRanker (deep learning model) scores particles
  ↓
[REFINEMENT STAGE] — Ab-initio reconstruction, non-uniform refinement
  ↓
[POSTPROCESS STAGE] — Motion & CTF refinement
  ↓
OUTPUT (High-resolution 3D volume, metadata)
```

### Major Components

1. **Core Orchestrator** (`CryoWizard.py`, `run.py`) — Entry point, 12+ operations, pipeline graph construction, job dependency management
2. **CryoRanker** — Vision Transformer (ViT-B, patch 14, 224×224), MoCo-v3 self-supervised training, v1.5 model weights
3. **Pipeline Stages** — Import → Preprocess → CryoRanker → Refine → Postprocess
4. **Job Management Layer** (`JobAPIs.py`) — Wraps cryosparc-tools API, 30+ job creation wrappers
5. **Configuration & Parameter System** — YAML global defaults + JSON per-node parameters + CLI overrides
6. **GUI** — Flask + Socket.IO web interface at port 39080; Chrome extension popup

### `run.py` (391 lines) — Execution Loop

**Key function `cryowizard_run()`**:
1. Loads base parameters, creates CryoSPARC handle
2. Reads pipeline.json DAG
3. Iterates steps (sorted by step number)
4. For each step: **build all nodes** first, then **execute all nodes**
5. Tracks best volume resolution across refinement steps
6. Outputs final `best_volume` to CryoWizard job

**Other functions**:
- `cryowizard_run_update_best_volume()` — Tracks lowest resolution (best) across jobs
- `cryowizard_kill()` — Recursively kills all child jobs

**Design Notes**:
- No transaction semantics — partial failure leaves orphaned jobs
- Best volume tracked via JSON file; no database
- Expensive I/O loop: reads job.json each iteration to extract resolution

### `refine.py` — 3D Reconstruction & Refinement

**`refine()` — Main orchestrator**:
1. Combines CryoRanker scores from all sources
2. Calls `refine_initial()` for ab-initio volumes
3. Runs `refine_search_single_turn()` × 3 iterations
4. Tracks best volume by resolution

**`refine_initial()` — Initial ab-initio phase**:
- `abinit_num` jobs (default 2)
- `base_abinit_particle_num` each (default 50k)
- Ab-initio → NU-Refine jobs in parallel
- Optional Orientation Diagnostics (CFAR/SCF)

**`refine_search_single_turn()` — Iterative refinement**:
- Creates grid of particle counts in search range
- For each grid point: Get particles by score → NU-Refine
- Tracks best by resolution; narrows range next turn

**Total GPU jobs**:
- Initial: 2 ab-initio + 2 NU-refine = 4 jobs
- Turn 0: 8 NU-refine jobs
- Turn 1: 4 NU-refine jobs
- Turn 2: 4 NU-refine jobs
- **Total: 20 GPU jobs per dataset** (brute force)

**Hardcoded thresholds**:
- `min_refine_score = 0.8` — only particles scoring ≥ 0.8 used
- Fixed 3 turns (no early stopping)
- No convergence criterion

### `vits.py` — Vision Transformer Model

**Model stats (ViT-Base)**:
- ~86M parameters (backbone) + ~few M (head)
- 224×224 input, 14×14 patches → 256 patch tokens + 1 CLS

## 4. Data Flow: End-to-End

```
RAW INPUT → [1] Import → [2] Motion Correction (if movies)
  → [3] CTF Estimation → [4] Exposure Curation
  → [5] Particle Picking (blob OR template)
  → [6] Particle Extraction → [7] Optional 2D Cleanup
  → [8] Particle Restack → [9] CryoRanker Scoring (ViT forward pass)
  → [10] Ab-initio (2 × 50k particles) → [11] Iterative Refinement (3 turns)
  → [12] Postprocessing (motion + CTF refine) → [13] Final 3D Volume
```

**CryoRanker score combination**:
1. Combine scores from all ranker jobs → single dict
2. Sort descending
3. "num" mode: top N; "score" mode: all ≥ threshold

**Metadata tracking per node**:
- `created_jobs_info.json` — [{jobuid, parents, gpu_need}]
- `output_groups.json` — {movie:[], micrograph:[], particle:[]}
- `type.json` — {parameters_type: "xxx"}
- `parameters.json` — full node parameters

## 9. Weaknesses & Gaps

1. **Hardcoded uncalibrated thresholds** — `min_refine_score=0.8` is not statistically justified
2. **No error recovery** — single job failure halts entire pipeline; no checkpointing
3. **Silent failures** — subprocess calls have no error handling; Slurm exit codes unchecked
4. **No audit trail** — parameters overwritten without timestamps; no decision log
5. **No convergence criterion** — fixed 3 turns regardless of improvement
6. **Contradiction in config** — `max_refine_score=0.1 < min_refine_score=0.8` (never used)
7. **Brute-force refinement** — 20 GPU jobs per dataset; no early stopping
8. **No parameter validation** — can set nonsensical values with no warnings
9. **Expensive resolution extraction** — parses job.json in a hot loop
10. **No reasoning trace** — impossible to audit why any decision was made
11. **No remote operation** — requires being on the GPU server subnet
12. **Dependency fragility** — cryosparc-tools must manually match CryoSPARC server version
