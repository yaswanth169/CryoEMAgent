## 2. System Architecture

### Top-Level Design

```
Input Particles (CryoSPARC .cs files)
    ↓
[Data Preprocessing & LMDB Creation]
    ↓
[CTF Extraction & Pose Extraction]
    ↓
CryoDECO Training:
├── Phase 1: Pretrain (GT poses, learn encoder+decoder)
├── Phase 2: Pose Search (hierarchical SO(3) grid, freeze encoder)
├── Phase 3: SGD Refinement (pose table + decoder)
└── Analysis: Clustering, PCA, volume generation
    ↓
Output: Clustered volumes + particle assignments + latent z
```

### Core Neural Network Components

#### A. Encoder

**CryosolverEncoder** — ViT backbone (vit_small or vit_base):
- 2D sincos position embeddings (fixed)
- Pre-trained from Cryo-IEF weights
- Outputs: z [batch, feature_dim] (default 128-dim)
- Fine-tuning strategies: `all` / `vit_block` (last N blocks) / `mlp_layer` / `random_initial`

**ConfTable** — Per-particle embedding table:
- Shape: [N_particles, conf_table_dim]
- Optimized via SGD during refinement phase
- Optional initialization from pre-trained features

**GatingMechanism** — Feature fusion:
```
gate = sigmoid(W_a @ z_pfm + W_b @ z_table)
z_fused = (1 - gate) * z_pfm + gate * z_table
```
- Types: 'gate' (multiplicative) or 'concat'
- Gate warmup + cosine annealing schedule (`fuse_only_table=0.35` → 0.0)

#### B. Pose Estimation

**Hierarchical Pose Search (HPS)**:
- SO(3) grid via healpy quaternion sampling
- Translation grid search
- Coarse-to-fine (l_min → l_max resolution)
- Gradient-based Lie algebra refinement
- Maintains top K poses per particle

**PoseTable** — Learnable per-particle poses:
- S2S2 representation: two 3-vectors → rotation matrix (normalize + cross product)
- Per-particle translation offsets
- Initialized from pose search, optimized via SGD

#### C. HyperVolume Decoder

**Architecture**:
```
Input: 3D coords [B, N, 3] + z [B, z_dim]
  ↓
Positional encoding (sinusoidal / Fourier / learned)
  ↓
Broadcast z + optional intermediate encoder features
  ↓
MLP/MOE/VitDecoder (n_layers deep, hidden_dim wide)
  ↓
Output: Hartley-space voxel density [B, N]
```

**Decoder types**: `mlp` (standard), `moe` (Mixture of Experts with learned routing), `vit` (transformer cross-attention)

**MOE**: `moe_num` expert branches, learned sparse routing (top-k per sample).

### `Pose/pose_search.py` — Hierarchical Pose Search

**`opt_theta_trans()`** — Full algorithm:

```
For l = [l_min, ..., l_max]:
  1. Sample SO(3) via healpy quaternions → rotation matrices
  2. For each R:
     x_rot = coords @ R
     y_pred = hypervolume.eval(x_rot, z, resolution=l)
     corr = sum(y_pred * particle)
  3. Keep top nkeptposes
  
  4. Translation search:
     For t in grid(-t_extent, +t_extent, t_n_grid²):
       y_trans = phase_shift(y_pred, t)
       compute correlation
     keep best t
  
  5. Gradient refinement via SO(3) Lie algebra:
     R_refined = exp(log(R) + δR)  where δR minimizes loss
```

```
  y_pred * CTF → loss vs y_gt
  Update: encoder + hypervolume
  ↓
[Phase 2: Pose Search]
  particles → ViT → z  (encoder frozen)
  opt_theta_trans(z, coords, hypervolume) → R_best, t_best
  HyperVolume(...) → y_pred → loss
  Update: hypervolume only
  ↓
[Phase 3: SGD Refinement]
  particles → ViT → z  (encoder frozen)
  pose_table[index] → R, t
  HyperVolume(...) → y_pred → loss
  Update: hypervolume + pose_table
  ↓
[Analysis]
  all particles → ViT → z_all [N, 128]
  UMAP(z_all) → 2D
  GMM/KMeans(z_reduced) → labels
  For each cluster:
    z_center = mean(z_cluster)
    vol = HyperVolume.generate_3D(z_center)
    save as .mrc
    save particle subset as .cs / .star
```

## 5. Algorithms & Mathematics

### CTF Physics Model
```
λ = 12.2639 / √(V + 0.97845e-6 * V²)           [de Broglie wavelength]
γ = 2π(-0.5 * df * λ * s² + 0.25 * Cs * λ³ * s⁴) - φ
CTF = √(1-w²) * sin(γ) - w * cos(γ)
```

### Variational Inference (optional)
```
z_sample = μ + √exp(logvar) * ε,  ε ~ N(0, I)
KL = 0.5 * sum(1 + logvar - μ² - exp(logvar))
L_total = L_recon + β_conf * KL
```

### Gating Mechanism Schedule
```
initial bias → table-favoring (0.35)
cosine annealing over training → balanced (~0.5)
```

## 7. Strengths

1. **Foundation model priors** — Cryo-IEF pretrained ViT avoids random-init bottleneck
2. **Dual heterogeneity** — single architecture for compositional + conformational
3. **Integrated pose estimation** — no pre-computed poses needed; hierarchical search
4. **Disentangled latent space** — pose vs. conformation cleanly separated
5. **Gated feature fusion** — learned optimal blend of encoder + table
6. **CryoSPARC end-to-end** — .cs input → .cs/.star output particle subsets
7. **Comprehensive visualization** — UMAP + PCA + traversal + annotated cluster plots
8. **Mixed precision + gradient checkpointing** — memory-efficient training
9. **Distributed training** — Accelerate (multi-GPU, multi-node)

---

## 8. Weaknesses & Gaps

1. **Depends on Cryo-IEF** — quality bottlenecked by pretrained model
2. **Computationally expensive** — pose search O(n_particles × n_rotations); 40GB GPU recommended
3. **k_num requires prior knowledge** — no auto-selection of cluster count
4. **No FSC output** — cannot validate reconstruction resolution directly
5. **No uncertainty quantification** — no confidence intervals on cluster assignments
6. **No symmetry support** — cannot exploit C2, C4, icosahedral symmetry
7. **Rigid body only** — no elastic/deformation modeling
8. **UMAP dim hardcoded** — 16 dimensions for clustering (not exposed as key param)
9. **Failure modes not characterized** — no guidance on when pose search fails
10. **No ablation studies** — comparison vs. random init, vs. encoder-only not shown
11. **Preprint only** — not peer-reviewed as of review date
