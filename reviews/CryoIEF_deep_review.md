## 1. Project Overview

**Cryo-IEF** (Cryo-EM Image Evaluation Foundation) is a comprehensive foundation model designed for cryo-electron microscopy (cryo-EM) image processing and analysis. It represents a major breakthrough in cryo-EM automation by providing general-purpose learned representations of cryo-EM particle images.

### Publication Details
- **Venue**: Nature Methods, November 2025
- **DOI**: https://doi.org/10.1038/s41592-025-02916-8
- **Paper Title**: "A comprehensive foundation model for cryo-EM image processing"
- **Authors**: Yang Yan, Shiqi Fan, Fajie Yuan, Huaizong Shen
- **Affiliation**: Westlake University (westlake-repl)

### Core Purpose
Cryo-IEF is pretrained on approximately **65 million cryo-EM particle images** using unsupervised learning via the MoCo v3 (Momentum Contrast) framework. The resulting foundation model excels at:
1. Particle structure classification (grouping particles by structural class)
2. Pose-based clustering (grouping particles by orientation)
3. Image quality assessment (ranking particles by reconstruction suitability)

## 3. Architecture Details

### 3.1 Backbone Model: Vision Transformer (ViT)

The foundation model uses a **Vision Transformer (ViT) with MoCo v3 modifications**, defined in `Cryo_IEF/vits.py`:

```python
class VisionTransformerMoCo(VisionTransformer):
    """Foundation model backbone with MoCo-specific modifications"""
    # 2D sin-cos position embeddings (not learnable, fixed)
    # Stop-grad patch embedding projection (optional)
    # Batch normalization layer options per transformer block
    # Xavier initialization for linear layers
```

**Supported Architectures** (configurable via `backbone_name`):
- `vit_small`: 384-dim, 12 layers, 12 heads, patch_size=14
- `vit_base`: 768-dim, 12 layers, 12 heads, patch_size=14 **(MOST COMMON)**
- `vit_large`: 1024-dim, 24 layers, 16 heads, patch_size=14
- `vit_huge`: 1280-dim, 32 layers, 16 heads, patch_size=14
- `vit_giant`: 1408-dim, 40 layers, 16 heads, patch_size=14
- `vit_conv_small`: ConvStem variant, 11 ViT blocks
- `vit_conv_base`: ConvStem variant, 11 ViT blocks
- `vit_hMLP_base`: hMLP stem variant

### 3.2 Input Processing

**Input Format**:
- Single-channel 2D cryo-EM particle images
- Typically normalized to **224×224 pixels** (configurable via `resize: 224`)
- Normalized to mean=0.5379, std=0.1180 (empirically determined for cryo-EM data)
- Batch size: **512** (configurable)

**Patch Embedding**:
- 14×14 patches (224×224 image → 16×16=256 patches + 1 CLS token)
- `Conv2d(1, 768, kernel_size=14, stride=14)` for ViT-Base

Defines the ViT backbone and all output heads (see §3 above). Critical for understanding the model internals. Key additions over standard ViT:
- Fixed 2D sin-cos positional embeddings (no learnable pos embed)

import subprocess, json

def cs_score_particles_with_cryoief(
    job_path: str,           # e.g. /cryosparc/J123_extract_particles

## 8. Strengths

1. **Massive pre-training scale**: 65M particles → robust, generalizable features
2. **Foundation model paradigm**: transfer learning to diverse tasks (ranking, clustering, reconstruction)
3. **CryoSPARC-native integration**: reads job folders directly
4. **Multi-GPU via accelerate**: scales to 8 GPUs, ~4000 particles/sec
5. **Pose clustering**: orientation analysis without 3D reconstruction (huge time saving)
6. **Feature compression**: float16 halves storage vs float32
7. **Modular**: CryoIEF / CryoRanker / CryoClustering can run independently or chained

---

## 9. Weaknesses & Gaps

1. **CryoSPARC-only input**: no direct raw MRC support; must preprocess first via cryodata
2. **Minimal RELION support**: only cs2star conversion at output; no RELION input pipeline
3. **No streaming feature export**: must pickle entire list (slow for >1M particles)
4. **No uncertainty/confidence output**: single point estimate, no per-particle confidence
5. **Score threshold not documented**: no guidance on what score → "good particle"
6. **Version complexity**: v1 vs v1.5 CryoRanker not clearly documented when to use which
7. **No error recovery**: if preprocessing fails midway, restart from scratch
8. **No progress checkpointing**: can't resume interrupted large-dataset inference
9. **Limited logging**: minimal debugging output; errors in preprocessing often silent
