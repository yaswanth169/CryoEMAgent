## 2. Architecture Overview

CryoDRGN-AI implements an **amortized variational inference framework** for heterogeneous reconstruction.

### Core Model (DrgnAI)

Three main components:

1. **Encoder** — Conformation estimation (optional CNN or lookup table)
2. **HyperVolume (Decoder)** — Continuous 3D volume generator conditioned on latent codes z
3. **Pose Table** — Per-particle rotation and translation parameters

### Complete Pipeline

```
Raw Cryo-EM Images
    ↓
Data Loading (MRC/STAR/CS files, CTF parameters)
    ↓
FFT → Hartley Transform → Symmetrization → Normalization
    ↓
Pose Search (SO(3) grid search) + Conformation Encoding
    ↓
Forward Pass:
  z = Encoder(image) or sample from table
  y_pred = HyperVolume(rotate(coords, R), z)
  y_pred *= CTF
    ↓
Loss: L2 frequency bias + KL divergence + translation regularizer
    ↓
Checkpoint + PCA + Clustering per epoch
    ↓
Output: Volumes + latent codes + poses
```

### `src/models.py` — Model Definition

**Class hierarchy**:
```
DrgnAI(nn.Module)
├── PoseTable: [n_particles, 3, 3, 2] rotation + translation
├── ConfTable: [n_particles, z_dim] latent codes
├── ConfRegressor: CNN encoder for amortized inference
│   └── SharedCNN → ConfTable
├── HyperVolume: MLP implicit neural representation
│   ├── Gaussian positional encoding
│   ├── MLP layers (n_layers deep, hypervolume_dim wide)
│   └── Output: real-valued Hartley-space voxels
└── PoseSearch: optimization-based pose refinement
```

**`DrgnAI.forward()`**:
1. `encode()` → z, R, t from image
2. `decode()` → rotate coords by R, query HyperVolume(x_rot, z), apply CTF
3. Return `{y_pred, y_gt_processed, ...latent_variables}`

**`encode()`**:
- `pose_only=True`: Random z (pretrain phase)
- Normal: `encoder(real_particles)` → z (amortized)
- `latent_optimization=True`: `conftable[i]` → z (autodecoder)
- Pose: GT lookup / grid search / pose table

**`decode()`**:
- Rotate lattice coords: `x_rot = coords @ R`
- Query HyperVolume(x_rot, z) → y_pred (Hartley space)
- Apply CTF: `y_pred *= ctf`
- Translate in Fourier space

### `src/pose_search.py` — Pose Optimization

**Algorithm** (`opt_theta_trans()`):

```
Phase 1: SO(3) Grid Search
for l = [l_start, ..., l_end]:
    quaternions = healpy_grid(base_healpy)  # HEALPix SO(3)
    for each R in rotations:
        x_rot = coords @ R
        y_pred = hypervolume.eval(x_rot, z)
        correlation = sum(y_pred * particle)
    keep top nkeptposes

Phase 2: Translation Search
for each top rotation R:
    for t in Cartesian grid (-t_extent to +t_extent, t_n_grid²):
        y_trans = phase_shift(y_pred, t)  # Phase shift theorem
        keep best t

Phase 3: Gradient refinement via SO(3) Lie algebra
```

**SO(3) sampling**: HEALPix icosahedral grid (base_healpy controls resolution).

**Key parameters**:
- `nkeptposes=8` — Top-K poses to keep
- `base_healpy=2` — HEALPix resolution (default very coarse for speed)
- `t_extent=20.0` — Translation extent in pixels
- `t_n_grid=7` — 49 translation grid points

### `src/ctf.py` — CTF Physics

```python
def compute_ctf(freqs, dfu, dfv, defocus_angle, volt, cs, w, phase_shift, b_factor):
    λ = 12.2639 / sqrt(volt + 0.97845e-6 * volt²)  # de Broglie wavelength
    γ = 2π(-0.5 * df * λ * s² + 0.25 * cs * λ³ * s⁴) - φ
    CTF = sqrt(1-w²) * sin(γ) - w * cos(γ)
```

## 4. Data Flow: End-to-End

### Training

```
INPUT: Particles (.mrcs/.star/.cs), CTF (.pkl), optional GT poses (.pkl)

1. Load: real space → Hartley transform → symmetrize → normalize
2. Batch: configurable size per phase
3. Encode: CNN or table → z [B, z_dim]
4. Pose: GT / grid search / table → R [B, 3, 3], t [B, 2]
5. Decode: rotate coords → HyperVolume(x_rot, z) → CTF → mask
6. Loss: L2 freq_bias + KL + translation regularizer
7. Checkpoint: weights.N.pkl, pose.N.pkl, conf.N.pkl

OUTPUT: Checkpoints + analysis_N/ directory
```

### Analysis

```
Checkpoint → z_all [N, z_dim] → PCA + UMAP → K-means/GMM
  → volumes (vol_*.mrc) + latent traversals + clustering plots
```

## 5. Algorithms & Models

### HyperVolume Architecture

```
Input: 3D coordinate [B, N, 3] + latent z [B, z_dim]
  ↓
Positional Encoding (Gaussian RBF): 
  pe(x) = [sin(2π σ₁ x), cos(2π σ₁ x), ...]  where σ ~ log-uniform
  ↓
Broadcast z to all N points → concatenate with pe(x)
  ↓
MLP: n_layers deep, hypervolume_dim wide, ReLU activations
  ↓
Output: density value [B, N] (Hartley-space)
```

### Loss Functions

```
L_total = L_recon + β_conf * L_kl + λ_trans * L_trans

L_recon = mean((resolution * freq_norm)^alpha * (y_pred - y_gt)²)
L_kl    = mean(sum((1 + logvar - μ² - σ²) / 2))
L_trans = mean(|t|)
```

### Frequency Marching Schedule

```
radius_t = radius_0 + n_images_processed / add_one_every
```
Prevents early overfitting; starts learning at low frequencies, gradually expands.

## 7. Strengths

1. **Continuous heterogeneity manifold** — learns smooth conformational space, no pre-classification needed
2. **Two inference modes** — amortized encoder (fast) + autodecoder (per-particle, accurate)
3. **Rigorous physics** — proper CTF, Hartley transform, frequency-weighted loss
4. **Frequency marching** — coarse-to-fine training, more stable optimization
5. **Cryo-ET support** — subtomogram averaging with tilt-series aware loss
6. **Comprehensive testing** — unit tests + fixture data + end-to-end test scripts
7. **TensorBoard logging** — real-time loss and latent space visualization
8. **Modular code** — clear separation of model/data/analysis/pose

---

## 8. Weaknesses & Gaps

1. **Deprecated** — now merged into cryoDRGN; this repo is no longer maintained
2. **Rigid body only** — no elastic/deformation modes
3. **Single particle per image** — no multi-body modeling
4. **No known-structure initialization** — can't use AlphaFold / existing map
5. **No symmetry** — no icosahedral, helical, or point-group symmetry
6. **Memory issues** — large hypervolumes can overflow GPU; no explicit management
7. **No learning rate schedule** — no warmup/decay; manual tuning required
8. **Limited multi-GPU** — DataParallel only, not DistributedDataParallel
9. **LBFGS slowness** — optional pose optimizer is expensive
10. **Dependency on cryoDRGN** — `from cryodrgn import ...` throughout
11. **Python 3.10 max** — 3.11+ not supported (`pillow < 10` constraint)
