## 1. Project Overview

**Package name**: `cryodata` (v0.1.3b02)

**Purpose**: Production-ready **data loading and preprocessing library** for cryo-EM single-particle analysis. Bridges raw CryoSPARC output and PyTorch training loops.

**Author**: Yang Yan (y.yan98@outlook.com), Westlake University

**Relationship to other projects**:
- **Primary consumer**: Cryo-IEF foundation model training (65M particles)
- **Also used by**: CryoDECO, CryoWizard, CryoRanker

**Core capabilities**:
1. Preprocessing pipeline — resize, normalize, window-mask particles from CryoSPARC
2. LMDB database creation — fast multi-process conversion, per-protein splitting
3. PyTorch Dataset/DataLoader integration with balanced samplers
4. Fourier representations — optional FFT/Hartley transform outputs
5. Format conversion — CryoSPARC .cs → RELION .star seamlessly
6. Label system — score source tracking (calculated vs. default vs. missing)

## 2. Architecture Overview

```
Raw cryoSPARC Data (MRC stacks + .cs metadata)
        ↓
raw_data_preprocess() [mrc_preprocess.py]
        ↓
    ├── Metadata loading & merging (cs_star_translate/)
    ├── MRC parsing and resizing (FFT-based for downsampling)
    ├── Normalization to uint8 (optional)
    ├── Hartley transform computation (optional)
    └── Storage: LMDB (recommended) or pickle
        ↓
    CryoMetaData [cryoemDataset.py]
        ├── Loads processed_data_path structure
        ├── Builds id→protein mappings
        ├── Loads labels (classification, clustering, score_source)
        └── Manages LMDB references
        ↓
    CryoEMDataset [cryoemDataset.py]
        ├── PyTorch Dataset interface
        ├── Lazy LMDB loading per worker
        ├── On-the-fly transforms
        └── Local crops, masking, pose-based mixup, MIM
        ↓
    DataLoader + Sampler [dataset_resample.py]
        ├── MyResampleSampler (fine-tuning: balanced pos/neg/mid)
        └── MyResampleSampler_pretrain (pre-training: all data)
        ↓
    Training Loop
```

### `data_preprocess/mrc_preprocess.py` (600+ lines)

#### Key Utilities

**`mrcs_resize(mrcs, width, height=None, is_freqs=True)`**:
- Frequency-domain: FFT crop for downsampling (sharp, no interpolation artifacts)
  ```python
  oldft = fft.ht2_center(imgs)
  newft = oldft[..., start:stop, start:stop]  # Crop in frequency space
  new = fft.iht2_center(newft)
  ```
- Spatial-domain: `scipy.ndimage.zoom(order=3)` for upsampling

**`mrcs_to_int8(mrcs)` / `to_int8(mrcdata)`**:
- Per-image normalization to [0, 255], convert to uint8 or PIL Image

**`window_mask(resolution, in_rad, out_rad=0.99)`**:
- Radial cosine-edge mask: 1.0 inside `in_rad`, tapers to 0.0 at `out_rad`
- Reduces edge artifacts before FFT

**`sample_and_evaluate(path_list, save_path, ...)`**:
- Samples random MRC files, estimates dataset statistics
- Outputs: `means_stds_raw.data`, `means_stds_FT.data`, `img_dim.data`

#### CryoSPARC Directory Processing

**`raw_csdata_process_from_cryosparc_dir(raw_data_path)`**:
- Locates .cs files in job directory
- Identifies particles: `*_particles.cs`, `*passthrough_particles.cs`, `*_split_0.cs`
- Merges primary + passthrough via inner join on UID
- Determines MRC directory: `restack/`, `extract/`, `imported/`, `downsample/`
- Returns: `(cs_data, mrc_dir)`

**`sort_csdata(cs_data, save_path)`**:
- Sorts particles by MRC filename + blob index
- Multiprocess sorting via `append_data`
- Returns: `(new_cs_data, mrcs_names_list, indices_dict)`

## 5. Augmentation Pipeline

### Preprocessing Transforms (in `raw_data_preprocess`)
1. Resize — FFT-based downsampling, bicubic upsampling
2. Window — Radial cosine-edge mask (in_rad=0.85)
3. Normalize — Per-image [0, 255], uint8
4. Optional Hartley transform

### On-the-Fly Augmentation (in Dataset)

Transform dictionary keys:
- `'ptcls'` — Primary particle transform (resize, normalize, ToTensor)
- `'mics'` — Micrograph transform
- `'random_resized_crop_all'` — Micrograph cropping
- `'random_rotate'` — In-plane rotation

**Advanced features**:
- **Dual augmentation** (`needs_aug2=True`) — for contrastive learning (aug1 + aug2)
- **Pose-based mixup** (`mix_pos_setting`) — Annoy index nearest-neighbor; `weight = sigmoid(sigma * ((3.5 - distance) / 3.5 - bias))`
- **Cross-slice augmentation** (`slice_setting`) — loads from alternate directory
- **Local crops** — `N` small random crops per image for multi-scale ViT training
- **MIM masking** (`mask_setting`) — random patch mask ratio from `uniform(mask_ratio[0], mask_ratio[1])`, binary mask shape `(W//patch_size, H//patch_size)`
- **Triplex labels** — score → {0.0 (neg), 0.5 (uncertain), 1.0 (pos)}

## 7. Strengths

1. **Efficient LMDB**: Per-protein splitting reduces contention; lazy per-worker env loading
2. **Robust labels**: Score source tracking (calculated/default/missing) + backward compatibility
3. **Sophisticated augmentation**: Dual aug, pose-based mixup (Annoy index), MIM, local crops
4. **Advanced sampling**: Score-source-aware resampling, per-protein + per-bucket balancing
5. **FFT-based resizing**: Sharp downsampling without interpolation artifacts
6. **CryoSPARC-native**: Direct job folder parsing, .cs → .star conversion
7. **Production ready**: `__del__` cleanup, safe loaders, multiprocess safe

## 8. Weaknesses & Gaps

1. **cryosparc_tools hard dependency** — cannot parse .cs without CryoSPARC installation
2. **LMDB reference manifest complexity** — all-or-nothing build, no incremental update
3. **Worker process management fragility** — manual worker ID tracking, potential FD leaks
4. **No data versioning** — no version field in preprocessed datasets
5. **No fault tolerance** — LMDB creation fails = restart from scratch
6. **Unused imports**: matplotlib, seaborn, scikit-learn, pyFFTW installed but not used
7. **No type hints** — no IDE autocompletion/type checking
8. **No test suite** — no test/ directory found
9. **Limited documentation** — minimal docstrings, no troubleshooting guide
10. **Score threshold not documented** — when is score "good"? No guidance.
