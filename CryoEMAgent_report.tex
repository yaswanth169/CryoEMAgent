\section{Introduction and Motivation}
\label{sec:intro}
% ════════════════════════════════════════════════════════════════════════════

\subsection{The Problem: Cryo-EM is Expert-Intensive}

Cryo-electron microscopy is one of the dominant techniques for high-resolution
protein structure determination. GPCR (G-protein-coupled receptor) structures
are of particular pharmaceutical interest — GPCRs represent $\sim34\%$ of all
FDA-approved drug targets — yet the computational pipeline for a single
structure determination run involves:

\begin{itemize}[leftmargin=1.5em]
  \item \textbf{19 sequential processing steps} across two sub-pipelines
        (W1 blob + W2 template), running for 3--8 hours on GPU hardware
  \item \textbf{Dozens of tunable parameters} per step (box size, NCC
        thresholds, number of 2D classes, regularisation parameters, etc.)
  \item \textbf{5 expert human checkpoints} requiring domain knowledge:
        CTF curation, particle pick inspection $\times2$, 2D class
        selection $\times2$
  \item \textbf{Remote GPU servers} — the scientist's laptop cannot run
        CryoSPARC directly; data and compute live on a separate server
\end{itemize}

\medskip
\begin{tcolorbox}[
  colback=yellow!8, colframe=checkpointyellow,
  arc=4pt, boxrule=0.8pt,
  title={\small\textbf{Core Challenge}}
]
\small
Each human checkpoint requires an expert to: (a) log into the CryoSPARC web
UI on the GPU server, (b) visually inspect CTF plots / particle picks / 2D
class images, (c) apply thresholds based on years of domain experience, and
(d) click ``Finish'' before the pipeline can continue. This bottleneck means
an overnight GPU run may sit idle for hours waiting for a 5-minute human
decision.
\end{tcolorbox}

\subsection{The Solution: CryoEMAgent}

CryoEMAgent provides an \textbf{autonomous agentic framework} that:

\begin{enumerate}[leftmargin=1.5em, label=\textbf{\arabic*.}]
  \item \textbf{Plans} the next pipeline action using a ReAct-pattern LLM
        with 15 formal CryoSPARC tool definitions
  \item \textbf{Executes} CryoSPARC jobs through Model Context Protocol (MCP)
        over SSH — operates from any laptop, anywhere
  \item \textbf{Assesses quality} via specialised critics (CTF, picking, 2D,
        refinement)
  \item \textbf{Evaluates checkpoints} using a VLM critic — auto-approves
        when confidence $\geq85\%$
  \item \textbf{Explains} every decision with a full reasoning chain
        (Observation $\to$ Thought $\to$ Tool $\to$ Decision)
  \item \textbf{Converses} naturally like Claude Code — scientists control it
        in plain English
\end{enumerate}

\section{Agentic Framework: Planning $\cdot$ Memory $\cdot$ Action}
\label{sec:agent}
% ════════════════════════════════════════════════════════════════════════════

The agent implements a formal \textbf{Planning $\to$ Memory $\to$ Action} loop
at every pipeline step, grounded in the ReAct framework~\cite{yao2023react}.

\subsection{Planning: ReAct Pattern}

At each step the LLM reasons through a structured chain:

\begin{tcolorbox}[
  colback=codebg, colframe=agentblue,
  arc=4pt, boxrule=0.7pt,
  title={\small\textbf{ReAct Reasoning Chain — example from live run}}
]
\begin{lstlisting}[numbers=none, frame=none, backgroundcolor=\color{codebg}]
Observation:  patch_motion completed. 20 micrographs produced.
              CTF estimation is the logical next step.

Thought:      Motion correction successful on all 20 micrographs.
              CTF estimation measures defocus per micrograph and
              is required before particle picking.

Tool:         run_patch_ctf_estimation

Decision:     CONTINUE

Reasoning:    Motion correction completed on all 20 micrographs.
              CTF estimation is mandatory before picking to
              characterise phase distortions for each exposure.
              No quality flags raised.

Recommendation: Proceed to CTF estimation.
\end{lstlisting}
\end{tcolorbox}

\medskip
\noindent
The LLM has access to \textbf{15 formally defined CryoSPARC tools} injected
into the system prompt:

\begin{table}[H]
\centering
\caption{Formal CryoSPARC tool definitions available to the LLM planner}
\label{tab:tools}
\small
\begin{tabular}{clp{7.5cm}}
\toprule
\textbf{\#} & \textbf{Tool name} & \textbf{Description} \\
\midrule

5  & \texttt{run\_blob\_picker}               & Laplacian-of-Gaussian particle detection \\

7  & \texttt{extract\_particles}              & Extract particle stacks (box=256 px for GPCR) \\
8  & \texttt{run\_2d\_classification}         & 50-class 2D averaging for quality assessment \\
9  & \texttt{select\_2d\_classes}             & \textbf{[CHECKPOINT]} Keep protein classes, discard junk \\
10 & \texttt{run\_abinit\_reconstruction}     & Ab-initio 3D model generation (no reference required) \\
11 & \texttt{run\_homogeneous\_refinement}    & Single-conformation 3D refinement \\
12 & \texttt{run\_template\_picker}           & Template-based picking using W1 2D class averages \\

\end{table}

\subsubsection{Decision framework}

\subsection{JSONL Framing}

Unlike typical MCP implementations that use \texttt{Content-Length} headers,
our server uses \textbf{newline-delimited JSON} (JSONL):

\begin{lstlisting}[caption={JSONL framing — request and response},
  label=lst:jsonl]
# Request (laptop -> GPU server)
{"jsonrpc":"2.0","id":1,"method":"tools/call",
 "params":{"name":"cs_continue_pipeline",
           "arguments":{"run_id":"f57bbb1e-...","steps":1}}}\n

# Response (GPU server -> laptop)
{"jsonrpc":"2.0","id":1,"result":
 {"step":"patch_motion","status":"running",
  "checkpoint":false}}\n
\end{lstlisting}

\noindent
Non-JSON lines (SSH banners, warnings) are silently skipped by the client.

\subsection{Windows Compatibility Fix}

A critical bug was found and fixed during development:

\begin{tcolorbox}[
  colback=red!5, colframe=codered, arc=4pt, boxrule=0.7pt,
  title={\small\textbf{Bug: OSError on Windows — \texttt{bufsize=0}}}
]
\small
\textbf{Root cause:} \texttt{subprocess.Popen(bufsize=0)} (unbuffered) is
incompatible with Windows pipe handles and raises
\texttt{OSError: [Errno 22] Invalid argument} on \texttt{stdin.write()}.\\[4pt]
\textbf{Fix:} Changed \texttt{bufsize=0} to \texttt{bufsize=-1} (default
system buffering) in \texttt{MCPStdioClient.\_\_init\_\_}.
Since \texttt{flush()} is called after every write, buffering mode does not
affect correctness.
\end{tcolorbox}

\section{Novel Contributions}
\label{sec:contributions}
% ════════════════════════════════════════════════════════════════════════════

\begin{enumerate}[leftmargin=1.5em, label=\textbf{\arabic*.}, itemsep=6pt]

  \item \textbf{ReAct-pattern LLM agent for cryo-EM.}
  Formal tool definitions (15 CryoSPARC tools), Observe $\to$ Think $\to$
  Act reasoning loop, and full decision transparency via JSON reasoning log.
  Prior cryo-EM automation (RELION autopick, cryoDRGN) is domain-specific
  without general-purpose LLM reasoning.

  \item \textbf{VLM-based checkpoint evaluation.}
  Two-tier automated visual quality assessment at the 5 human-required steps
  using GPCR-specific prompts. Tier 1 (metrics reasoning) works over SSH;
  Tier 2 (image inspection) uses GPT-4V or Claude Vision. Auto-approves
  at $\geq85\%$ confidence, eliminating the human bottleneck for clean data.

  \item \textbf{MCP-over-SSH architecture.}
  Laptop-to-GPU-server agentic communication using the same JSON-RPC 2.0
  protocol as Cursor / Claude Desktop, enabling remote autonomous operation
  without a VPN or dedicated orchestration server.

  \item \textbf{Dual-mode UX (Autopilot + Interactive).}
  Same MCP engine, two interfaces: autopilot (fire-and-forget overnight runs)
  and interactive (conversational like Claude Code). Researchers with different
  expertise levels can choose their interaction style.

  \item \textbf{W1\,+\,W2 two-wave pipeline.}
  The W1 blob pipeline bootstraps a reference-free ab-initio model;
  the W2 template pipeline uses those 2D averages as picking templates for
  improved particle yield and purity — fully automated by the agent.

\end{enumerate}

\section{Limitations and Future Work}
\label{sec:limitations}
% ════════════════════════════════════════════════════════════════════════════

\begin{itemize}[leftmargin=1.5em, itemsep=4pt]

  \item \textbf{VLM Tier 2 image access:} Image-based checkpoint evaluation
        (Tier 2) currently requires local CryoSPARC access. Remote image
        download via MCP is planned for v0.3.

  \item \textbf{Mid-run parameter adjustment:} Dynamic parameter updates
        (e.g., increasing the number of 2D classes mid-run) require a restart.
        In-flight parameter patching is under development.

  \item \textbf{Truly zero-intervention 2D selection:} Automated 2D class
        selection AI (TOPAZ-style neural network or CLIP-based class ranker)
        would eliminate the two \texttt{select2d} checkpoints entirely.
        This is identified as the next research step.

  \item \textbf{Multi-dataset adaptation:} The GPCR-specific thresholds
        (particle diameter, symmetry, resolution target) are hardcoded.
        A dataset-adaptive mode that infers these from the data is planned.

  \item \textbf{VLM metrics sourcing:} Current Tier 1 evaluation uses
        simulated metrics (no live CryoSPARC metrics API). Live CTF and
        particle statistics from the CryoSPARC REST API would improve
        assessment accuracy.

\end{itemize}

\section{Conclusion}
\label{sec:conclusion}
% ════════════════════════════════════════════════════════════════════════════

CryoEMAgent v0.2 demonstrates that an LLM-based agentic framework can drive
a complete cryo-EM GPCR structure determination pipeline with minimal human
intervention. The key contributions — ReAct-pattern planning with formal tool
definitions, VLM-based checkpoint evaluation, and MCP-over-SSH remote
operation — address the three primary bottlenecks in current cryo-EM
workflows: reasoning transparency, quality-gate automation, and remote
operability.

The system was validated end-to-end on EMPIAR-10288, completing all 19
pipeline steps (J188--J206) with 5 human reviews. The architecture is
extensible to other cryo-EM targets, other structure determination
software (RELION, EMAN2), and other agentic frameworks.
